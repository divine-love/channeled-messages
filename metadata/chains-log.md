# Candidate Thought-Chains — Working Notes

## Purpose

People treat these messages as scripture. A chain layer that traces a single
idea across the archive — in the order the angels gave it, with each message
read in full — is both a study aid and a safeguard against the teachings being
quoted in fragments out of context.

This document is where we NOTICE candidate chains as we process each message.
We are not building chains yet. We are gathering evidence so that when the
archive is complete (messages uploaded through 2026), the eventual chains can
be built once, against the full set, with the entire arc visible.

## Design decisions already made (for future reference)

- **Unit of linking:** the whole message (message-level anchoring), never a
  sentence or fragment. This forces whole-context reading and discourages
  proof-texting.
- **Structure within a chain (revised 2026-07-06):** a chain is organized as
  role SECTIONS, in the order the argument requires (the registry's
  "argument it traces" column is the spine), with each section holding ALL
  its qualifying messages in chronological order by `date` received.
  Rationale: the angels rephrase important teachings because they are not
  always absorbed the first time, so restatements are part of the revelation
  pattern, not noise — strong witnesses are included, not discarded, and
  multiple messages may share a role. Within each section, one message is
  designated the ANCHOR (the developed statement); the others stand beneath
  it as witnesses, chronologically. Chronology thus still carries the
  angels' intended ordering, one level down: argument order lives in the
  sections, timing lives within each section.
- **A chain is a PATH, not a BAG.** A collection is unordered membership
  ("all the Two Paths messages"). A chain is an ordered argument that builds
  ("Hell is not eternal -> the dark spheres are a condition not a sentence ->
  any soul there can still turn to God -> therefore no soul is beyond
  redemption"). If a proposed chain is just "every message tagged X," it is a
  collection, not a chain. The anchor-per-section rule is the guard that
  inclusion never collapses a path back into a bag: a section may hold many
  witnesses, but the path is walked anchor to anchor.
- **One message belongs to many chains.** Chains will be defined in their own
  files (a future `chains/` directory, one file per chain, membership only).
  Chains are NEVER stored in a message's own front matter. In-message
  "next/previous in this chain" navigation will be GENERATED by a script that
  inverts the chain index — so bidirectional navigation costs nothing extra to
  maintain once the chain files exist.
- **Terminology:** prefer "paths" or "doors" over "chain" in any
  reader-facing language, to avoid collision with the existing "Chain of
  Light / Lattice of Light" terminology. ("Chain" is fine internally.)
- **Build timing:** construct the real chains AFTER the archive is caught up
  through 2026. Until then, only notice and log here.

## Link-role vocabulary (controlled — use these exact terms)

When logging a message against a candidate chain, tag what role it plays in
that chain's argument:

- **Foundation** — establishes the premise the rest of the thread depends on.
- **Elaboration** — deepens or extends a claim already made.
- **Objection-removed** — clears a contrary belief that blocks the next step.
- **Reframe** — recasts an idea in a new light or corrects a misunderstanding.
- **Testimony** — a spirit's first-person experience offered as evidence.
- **Chrysalis** — the transformational turn: once an objection is cleared or a
  premise laid, the chain opens a glimpse of what God's Divine Plan holds for
  the soul who walks this path. Forward-looking and revelatory. Distinct from
  Reframe (which changes how an idea is *seen*) and Testimony (which offers
  *evidence*); the Chrysalis shows the transformed vista the cleared ground
  now makes visible — the brick wall turned to glass.
- **Capstone** — the conclusion the thread builds toward. Distinct from
  Chrysalis: a Capstone *closes the argument*; a Chrysalis *opens a vista*. A
  chain may end on either, or carry both.

A message may carry different roles in different chains.
